## How to Use This Guide

1. Read sprint notes **in order** — each builds on the previous.
2. Every sprint note includes interview questions — use them for self-assessment.
3. The "Why we built it" sections explain architectural reasoning, not just implementation.
4. Cross-reference with [ARCHITECTURE.md](./ARCHITECTURE.md) for the system-level view.
